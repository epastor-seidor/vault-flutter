## Why

El proyecto DevVault carece de documentación en español para desarrolladores y usuarios. El README actual es boilerplate genérico de Flutter sin información sobre qué hace la app, cómo se estructura, cómo se ejecuta o cómo se contribuye. Esto dificulta el onboarding de nuevos colaboradores y la comprensión del sistema.

## What Changes

- Crear README.md completo en español con: descripción, características, tech stack, instalación, ejecución, estructura del proyecto, y guía de contribución
- Crear documento de arquitectura (`docs/arquitectura.md`) explicando el flujo de datos, proveedores Riverpod, modelo de encriptación y estructura de pantallas
- Crear guía de desarrollo (`docs/guia-desarrollo.md`) con instrucciones para configurar el entorno, convenciones de código, y cómo agregar nuevas funcionalidades
- Crear guía de usuario (`docs/guia-usuario.md`) explicando cada sección de la app (credenciales, notas, tareas, ajustes)

## Capabilities

### New Capabilities
- `project-documentation`: Documentación completa del proyecto en español cubriendo instalación, arquitectura, desarrollo y uso
- `developer-guide`: Guía técnica para desarrolladores con convenciones, estructura y flujo de trabajo
- `user-guide`: Manual de usuario explicando todas las funcionalidades de la aplicación
